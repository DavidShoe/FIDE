PROGRAM

Description

Defines a program that can be executed on the 9010A, either programmatically, or via user control.

You can use a number or a logical name to define your program. A logical name is recommended.
  • Program numbers must be decimal numbers in the range of 0 - 99.

  • Avoid using byte counts. This is a function of the legacy compiler, and is not necessary.

  • Symbolic names can be used, and are preferred.

  • The source file can contain no more than 100 9010A programs.

  • Numbered programs must appear in the correct order. If programs with symbolic names are combined with numbered programs, there must be a correct numbre of symbolically named programs between numbered programs. (Really, use symbolically named programs unless you have a good reason not to.)

Example

program OneTest
declarations
   assign REG1 to InputAddr
!! .. program starts
