INCLUDE

Description

Used to include another program or code segment in your current program.

FIDE Enhancement: One of the important enhancements FIDE implements is the ability to include relative path and long file names. A few notes about the include directive:
  • There are finite limits to the depth of the includes (legacy compiler limit).

  • Use "~/" to indicate that the include is relative to your current project directory. An include of "~/lib/Test.9lc" would translate to a physical path of "C:\Your Scripts\Lib\Test.9lc" if your project root was "C:\Your Scripts".

  • If you do not use the "~/" specifier, then the include paths are relative to your current location.

  • Pod includes are handled automatically Any include file ending with ".pod" is assumed to be a built-in pod include file.

  • You cannot include two files of the same name, even if they exists in two directories. I.e. you cannot include "~/lib/FileA.9lc" and "~/util/FileA.9lc" in the same script. The program will not compile properly.

  • You can include a file anywhere in your script.

Example

include "6502.pod"
include "~/lib/General Lib.9lc"
include "/utils/RAM Utils.9lc"
